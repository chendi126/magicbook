# 🚀 快速开始指南

## 📋 前置要求

1. **开发环境**
   - DevEco Studio 4.0 或更高版本
   - HarmonyOS SDK API 10 或更高版本

2. **测试设备**
   - 鸿蒙真机设备，或
   - DevEco Studio 内置模拟器

---

## 🛠️ 项目配置

### 1. 打开项目
```bash
# 使用 DevEco Studio 打开项目目录
File -> Open -> 选择 magicbook 文件夹
```

### 2. 同步依赖
项目打开后，DevEco Studio 会自动同步依赖。如果没有自动同步，可以手动触发：
```
File -> Sync Project with Gradle Files
```

### 3. 配置签名（真机调试需要）
- 在 DevEco Studio 中配置自动签名
- 或手动配置签名文件

---

## ▶️ 运行应用

### 方法1：使用模拟器
1. 点击工具栏的设备选择器
2. 选择或创建一个鸿蒙模拟器
3. 点击运行按钮（绿色三角形）

### 方法2：使用真机
1. 连接鸿蒙设备到电脑
2. 在设备上开启开发者模式和 USB 调试
3. 在 DevEco Studio 中选择设备
4. 点击运行按钮

---

## 📱 功能测试

### 测试1：图片取色
1. 启动应用，默认在"提取"页面
2. 点击"选择图片"按钮
3. 从相册选择一张色彩丰富的图片
4. 等待2-3秒，查看提取的配色
5. 尝试点击"复制 HEX"或"复制 RGB"按钮

### 测试2：保存配色方案
1. 在提取配色后，点击底部"保存配色方案"按钮
2. 在弹出的对话框中输入名称，如"夏日海滩"
3. 点击"保存"
4. 应用会自动切换到"配色库"页面

### 测试3：管理配色库
1. 切换到"配色库"标签
2. 查看已保存的配色方案
3. 点击某个方案的"删除"按钮
4. 在确认对话框中点击"删除"

## 🐛 常见问题

### Q1: 选择图片后没有反应？
**A**: 检查是否授予了相册访问权限。在设备的设置中找到应用，手动授予权限。

### Q2: 颜色提取失败？
**A**: 可能的原因：
- 图片格式不支持（建议使用 JPG/PNG）
- 图片文件损坏
- 查看控制台日志获取详细错误信息

### Q3: 保存失败？
**A**: 检查数据库是否初始化成功。查看应用启动日志。

### Q4: 配色库显示为空？
**A**: 
- 确认已经保存过配色方案
- 尝试重启应用
- 检查数据库文件是否存在

## 📊 性能建议

1. **图片大小**
   - 应用会自动将图片降采样到 800x800
   - 建议选择中等大小的图片（1-5MB）

2. **颜色数量**
   - 默认提取 6 种颜色
   - 可以在代码中调整（ColorExtractor.extractFromImage 的第二个参数）

3. **采样率**
   - 默认每隔 10 个像素采样一次
   - 可以在 ImageUtils.samplePixels 中调整

## 🔍 调试技巧

### 查看日志
在 DevEco Studio 的 Hilog 窗口中查看日志：
```
标签筛选：
- ColorExtractor: 颜色提取相关
- DatabaseManager: 数据库操作相关
- ImageUtils: 图片处理相关
```

### 常用日志
```typescript
// 图片选择
console.info('选择图片: ' + uri);

// 颜色提取
console.info('采样了 X 个像素');

// 数据库操作
console.info('保存配色方案成功，ID: X');
console.info('查询到 X 个配色方案');
```
